# Idea Killing

Find the fastest, cheapest way to disprove any idea. Ideas that survive are 10x more likely to work.

## The kill protocol
For every idea, answer:
1. **Cheapest kill test**: What's the fastest experiment that would prove this idea wrong?
   - Can you check in <5 minutes? (search, read data, ask one person)
   - Can you check in <1 hour? (build a prototype, run a test)
   - Can you check in <1 day? (talk to users, analyze data)
2. **Strongest counterargument**: What's the best case AGAINST this idea?
3. **Most likely failure mode**: How does this idea die in practice?
4. **Who wouldn't want this?**: Name a specific person in your target audience who would NOT want this. Why?

## Kill criteria
An idea should be killed if:
- The cheapest kill test takes >1 day AND the idea is unproven
- The counterargument is stronger than the argument
- The failure mode is the default outcome (most ideas fail by neglect)
- >50% of target users wouldn't want it

## Anti-pattern
Don't kill ideas because they're hard. Kill them because they're wrong.
"This is hard to build" is not a kill. "Nobody wants this" is a kill.

## Output
For each idea: kill test + counterargument + failure mode + verdict (kill/survive/needs data)
