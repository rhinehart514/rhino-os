# Pattern Theft

Study successful products. Steal their PATTERNS, not their features.

## How to steal patterns
1. Pick 3-5 successful products (any domain)
2. For each: what's the ONE structural pattern that makes it work?
   - Notion: everything is a block (composability)
   - Figma: real-time multiplayer (collaboration as default)
   - Stripe: docs ARE the product (developer trust through transparency)
   - Linear: opinionated defaults (speed over flexibility)
   - Superhuman: speed as the feature (performance IS the product)
   - Obsidian: local-first (ownership as differentiator)
   - Vercel: deploy preview per PR (confidence through preview)
3. Apply the pattern to our product:
   - "What if everything in rhino-os was composable like Notion blocks?"
   - "What if rhino-os was opinionated like Linear — fewer options, faster?"

## Pattern library (use WebSearch to find more)
- Composability (Notion, Unix pipes)
- Real-time collaboration (Figma, Google Docs)
- Docs as product (Stripe, Tailwind)
- Opinionated defaults (Linear, Rails)
- Speed as feature (Superhuman, SQLite)
- Local-first (Obsidian, Git)
- Preview before commit (Vercel, Storybook)

## Output
3 patterns applied to our product. Each with a specific implementation idea.
