# rhino-os → [founder-os](https://github.com/rhinehart514/founder-os)

> **rhino-os has been superseded by [founder-os](https://github.com/rhinehart514/founder-os).** All active development continues there.

founder-os is the full lifecycle version: ideate → validate → decide → build → measure → ship → learn. Everything rhino-os did (scoring, eval, taste, go, plan, retro) plus ideation, validation, portfolio management, proactive market intelligence, and 2026-aware business judgment.

## Migrate

```bash
claude plugin add rhinehart514/founder-os
```

All build/measure features are preserved. Skills renamed: `/rhino` → `/founder`, `rhino.yml` → `founder.yml`, `bin/rhino` → `bin/founder`.

New features in founder-os:
- **Ideation phase** — `/discover`, `/ideate`, `/decide`, `/portfolio`
- **Validation phase** — `/research`, `/strategy`, `/product`, `/money` with live market data
- **Proactive intelligence** — WebSearch + Playwright check competitors and markets before every planning session
- **Feature purpose audits** — `/retro audit` checks if features still matter as markets shift
- **14 failure patterns** — portfolio sprawl, revenue avoidance, research addiction, and 11 more
- **2026 context** — outcome-based pricing, MCP as infrastructure, open source at parity

## What rhino-os was

Product quality measurement for vibe-coded products. A Claude Code plugin that scored your product continuously, found bottlenecks, built autonomously, and learned across sessions. 27 skills, 14 agents, prediction-driven learning loop.

rhino-os measured products. founder-os is a cofounder — it validates ideas before you build, measures while you build, and learns across ventures.
