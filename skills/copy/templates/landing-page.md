# Landing Page Structure

## Hero Section

```
[HEADLINE — 7 words max]
[Subhead — 15 words max. Names the person. States what changes.]
[CTA button]
```

## Problem Section

```
[2-3 sentences in customer language]
[The pain they recognize — use their words]
[What they've tried and why it doesn't work]
```

## Solution Section

```
[What your product does — specific, not abstract]
[How it works — one visual or demo]
[What's different — vs the thing they're using now]
```

## Proof Section

```
[Customer quote — real, attributed]
— or —
[Metric — specific, verifiable]
— or —
[Empty — "No proof yet" is fine at stage one]
```

## CTA Section

```
[Repeat CTA with different framing]
[What happens next — set expectations]
[Objection handling — address the #1 reason they'd hesitate]
```

## Rules

- One page, one action. Don't split attention.
- Above the fold: headline + subhead + CTA. Nothing else required.
- Every section answers one question. Problem: "Do you have this pain?" Solution: "Here's what changes." Proof: "Others agree." CTA: "Do this now."
- No navigation links that lead away from the CTA.
- Mobile-first. If it doesn't work at 390px, it doesn't work.
