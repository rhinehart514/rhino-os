# Discovery Gotchas

- **Systems-first thinking**: Designing perfect systems before understanding if anyone wants them. Start from the user.
- **Analysis paralysis**: Discovering is comfortable. Building is uncomfortable. Set a time limit.
- **Discovering what you know**: Re-analyzing familiar territory because it feels productive. Focus on Unknown Territory.
- **Scope creep through discovery**: Every session finds "just one more thing." Scope expands. Resist.
- **Dependency over-mapping**: Not every connection is a dependency. Map the critical path, not the full graph.
