# Product Reasoning (2026)

## Reasoning Order (always follow this sequence)
1. **JTBD** → What job is the user hiring this for? What are they firing?
2. **Value prop** → Which value mechanism? Who benefits? What friction removed?
3. **Disruption check** → Sustaining (incumbents win) or disruptive (new market / underserved)?
4. **Workflow impact** → Which workflow touched? Faster/more reliable? Adjacent workflows broken?
5. **Moat check** → Does this generate proprietary data, network effects, or context depth? If none → commoditizable.
6. **Feature behavior** → User sees what? Inputs, outputs, empty states, failure modes
7. **Eval plan** → Which value proxy moves? 3x rule pass? Which perspective breaks it?
8. **Implementation** → Now write code

## Value Mechanisms
| Mechanism | 2026 AI-Native Version |
|-----------|----------------------|
| Time compression | AI does it FOR you |
| Quality uplift | AI output exceeds what user could do alone |
| Reach | AI finds the right audience automatically |
| Engagement | AI creates personalized, contextual nudges |
| Aliveness | AI generates ambient activity at low density |
| Loop closure | AI anticipates next step, pre-fills, auto-routes |
| New capability | Impossible without AI |
| Coordination reduction | AI mediates and resolves without humans |

## Stage-Aware Filters
- **Pre-launch**: creation > consumption, single-node density, core loop completion, MVI (users expect AI reasoning from day 1)
- **Growth**: retention > acquisition, loop closure, viral conversion, proprietary data flywheel
- **Mature**: efficiency, platform, defensibility, decision traces as moat

## Moat Assessment (2026)
Not a moat: using AI, having features, better UX (all commoditized)
IS a moat: proprietary data flywheels, decision traces, network effects + AI, context engineering depth, business model innovation

## Anti-Patterns (instant reject)
- Requires more users than product currently has
- Builds consumption before creation when creation is bottleneck
- Screens without outbound links
- Optimizes metrics before core workflow completes
- Builds infrastructure before product is proven
- Sustaining innovation competing against incumbents with more resources
- AI feature that fails the 3x rule (value < 3x compute cost)
- No clear JTBD (building for a job nobody has)

## Kill Criteria
- No real user need in 30 days → pause
- Can't name ONE person who'd pay → rethink
- Core loop "almost done" for >2 months → something's wrong
- Building to avoid selling → stop building

## Disruption Test (Christensen)
AI is NOT inherently disruptive. Business model determines disruption.
- **New-market**: Serving non-consumers (no current solution) → startup advantage
- **Low-end**: Radically cheaper/more accessible than incumbents → startup advantage
- **Sustaining**: Making existing product better → incumbents win, don't compete here

## The 3x Rule
AI feature must create ≥3x its compute cost in user value. If $0.15/run doesn't save $0.45+ → it's research, not product.
