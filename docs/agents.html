Decision-changing findings

Outputs that changed what to build, what to kill, or how to position. This is the only thing that matters.
SCOUT Flare has 500K users across 500+ colleges
Changed competitive positioning from "no student-first tools exist" to "no student-first creation tools exist"
Scout found Flare owns campus coordination. This means competing on events/RSVPs is suicide. Creation is the only open lane. This single finding reframed the entire product strategy.
DESIGN 1,684 arbitrary text sizes bypass the semantic type scale
Identified the #1 design debt item across 393 components that code review would never catch
A 10-token semantic type system exists in tokens.css but almost nobody uses it. text-[15px], text-[13px], text-[11px] scattered everywhere. One Tailwind config change + find-replace would make the entire app feel 2x more coherent.
BUILDER TypeScript has 3,406 error lines that sweep missed
Builder ran independent checks instead of parroting sweep, found a real issue
All errors in whitelist-manage.ts. Sweep checked "build passes" (it does — turbo caches) but never ran tsc. Builder's gate checklist caught what the daily triage missed.
META 3 of 5 agents were silently dead for 2+ days
Self-healed the system: identified CLAUDECODE nesting bug, fixed it, verified all agents operational
Sweep, builder, design-engineer all crashed on claude -p nesting. Nobody noticed because the LaunchAgents ran silently. Meta diagnosed the root cause (CLAUDECODE env var), applied the fix, and verified with live runs.
