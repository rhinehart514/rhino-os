# Todo Management Gotchas

- **Todo list as procrastination**: Capturing todos feels productive. Building them is productive. A growing backlog with nothing done is a warning sign.
- **Decay threshold arbitrariness**: 7/14/30 day thresholds are guesses. Right values depend on project velocity.
- **Smart promote mis-ranking**: Ranking by weight × is_bottleneck misses effort estimation. An easy todo on a non-bottleneck might deliver more per-hour.
- **Graduation false positives**: Similar keywords don't mean same problem. "Fixed auth bug" and "Fixed auth redirect" are different issues.
- **Source matters**: Todos from /eval vs /go vs founder have different priority implications.
- **Never killing**: Some todos should die, not be promoted. 60+ days in backlog with nobody caring = delete.
- **Stale without decay**: If decay detection isn't running, todos accumulate silently.
