# Constraint Thinking

Add artificial limitations. Constraints eliminate the obvious and force creativity.

## Constraints to try
- **No UI**: Build this with zero visual interface
- **5 seconds**: User must get value in 5 seconds
- **$0 budget**: No paid APIs, no hosting costs
- **100 tokens**: Entire interaction in 100 tokens
- **One file**: Entire product in one file
- **No code**: Solve this without writing any code
- **$1000/month price**: What would make this worth $1000?
- **Competitor copies you tomorrow**: What's left that they can't copy?
- **10-year-old user**: Simple enough for a child
- **Offline only**: No internet connection

## Method
1. Pick 2-3 constraints
2. For each: what solution emerges ONLY under this constraint?
3. The $1000/month constraint reveals your real value prop
4. The "competitor copies you" constraint reveals your moat

## Output
One solution per constraint. Note which constraints produced the most surprising solutions.
