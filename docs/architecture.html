System architecture

Four layers. Programs think. Agents act. Scoring measures. Knowledge remembers. The filesystem connects everything.

The stack

Top to bottom: brain → hands → senses → memory. Each layer reads from the one below and writes results down.

Layer 1 — Programs (the brain)

strategy.md program

Analyzes portfolio, checks escape velocity, picks what to build. Outputs sprint plan to active-plan.md.

build.md program

The one loop. Auto-detects scope, executes, measures, keeps or discards. 21KB of decision logic.

gate plan build experiment doctor

meta.md program

Self-evaluation. rhino-os improves rhino-os. Checks prompt sizes, freshness, stale plans.

Inter-agent communication — filesystem as message bus

No MCP server in the hot path. Agents read and write files. The filesystem IS the IPC layer.

FromToViaWhat
strategist builder .claude/plans/active-plan.md Sprint plan with prioritized tasks
sweep all agents .claude/state/sweep-latest.md Classified findings (GREEN/YELLOW/RED/GRAY)
scout strategist ~/.claude/knowledge/landscape.json Market positions with evidence
score.sh builder, strategist .claude/scores/history.tsv Per-dimension scores over time
taste.mjs builder, design .claude/evals/reports/taste-*.json Visual evaluation with evidence
any agent future sessions ~/.claude/knowledge/taste.jsonl Taste signals (deduplicated, strength-promoted)
builder strategist CLAUDE.md (target project) Updated eval state, sprint priority, gaps
why filesystem, not MCP?

MCP server exists but isn't in the hot path. Direct filesystem reads save tokens — an agent reading a JSON file costs zero API calls vs. an MCP tool call that consumes input/output tokens. The MCP server is there for external integrations, not inter-agent communication.

symlink architecture

Source of truth lives in the git repo. install.sh creates symlinks into ~/.claude/. This means git pull updates all agents/skills/programs immediately. Knowledge and state are NOT symlinked — they're user data that persists independently.

Event hooks — automation triggers

Hooks fire on Claude Code events. They extend the system without modifying agents.

enforce_ideation_readonly.sh
PreToolUse
Blocks file edits during ideation/gate mode. Prevents premature implementation.
track_usage.sh
PostToolUse
Logs every tool call to usage.jsonl. Enables focus drift detection.
capture_knowledge.sh
Manual
