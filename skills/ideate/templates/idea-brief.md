# Idea Brief Template

Copy this structure for every idea. Don't skip fields — empty fields mean the idea isn't ready.

```
▸ **[Idea Name]** — from [evidence source]

  evidence: [What data says this is the right idea? Be specific — cite scores, predictions, quotes]

  what: [3-5 sentences. Walk through what changes for the user. Not technical — behavioral.]

  who: [Name the person AND their situation. "A solo founder who just..." not "users."]

  changes: [Which metric moves? Which sub-score? Which assertion passes? Be specific.]

  costs: [What gets deprioritized? How long? What breaks?]

  kills it: [The one thing that would prove this idea wrong. The cheapest test.]

  draft assertions:
    - [mechanical assertion 1]
    - [mechanical assertion 2]
```

## Quality checks
- If `evidence` is empty or vague → idea isn't ready
- If `who` says "users" → too vague, name the person
- If `changes` has no number → not measurable
- If `kills it` is missing → you haven't thought hard enough
- If assertions are all `llm_judge` → prefer mechanical checks
