# Strategy Frameworks

Read on demand when the model needs framework context. Not loaded by default.

## Stage-Appropriate Work

Every stage has ONE question. Work that doesn't answer it is waste.

### Stage one (0 users)
**Question:** Can one person get value from this?
**Focus:** One feature, full depth. First loop works end-to-end.
**Acceptable:** Low craft, rough edges, manual processes, ugly UI.
**Anti-pattern:** Polishing (craft > delivery + 15). Building feature #4 before feature #1 scores 70. Growth features. Pricing optimization.
**Graduation:** 1 external user completes the full loop.

### Stage some (1-10 users)
**Question:** Do they come back?
**Focus:** Retention. Polish the core loop. Fix what breaks for real users.
**Acceptable:** Feature gaps, imperfect onboarding, limited docs.
**Anti-pattern:** New features instead of retention fixes. Ignoring user feedback. Building for users you don't have.
**Graduation:** 3+ users retain week-over-week.

### Stage many (10-100 users)
**Question:** How do new users find this?
**Focus:** Distribution. Onboarding. Word of mouth mechanics.
**Acceptable:** Minor UX issues, incomplete docs, some edge cases.
**Anti-pattern:** Features over distribution. No channel strategy. Building without knowing how users arrive.
**Graduation:** Consistent organic growth without founder-driven distribution.

### Growth (100+ users)
**Question:** Does the business work?
**Focus:** Unit economics. Revenue. Churn reduction.
**Acceptable:** Edge case gaps, non-critical bugs.
**Anti-pattern:** Revenue avoidance. Ignoring churn signals. Building features nobody asked for.
**Graduation:** Sustainable revenue covers costs.

## Bet Scoring Dimensions (7)

Each scored 1-10. Anti-sycophancy gate: minimum 2 dimensions MUST score below 5.

1. **Market gap** (1=packed, 10=greenfield) — Count funded competitors. Check for traction, not just funding.
2. **Timing** (1=too early/late, 10=perfect) — What technology or market shift enables this NOW? No shift = score 3.
3. **Founder-market fit** (1=no relevant skills, 10=perfect fit) — Read codebase and past work. Be honest about gaps.
4. **Distribution** (1=no channel, 10=proven) — Name the channel. Can't name it = score 2 max.
5. **Business model** (1=unclear, 10=proven) — What do competitors charge? What's the value metric? No answer = score 3.
6. **Defensibility** (1=weekend project, 10=deep moat) — Network effects, data, switching cost, or speed. Honesty matters here.
7. **Evidence quality** (1=pure intuition, 10=validated) — Count citations. No evidence = score 2.

## Sub-Score Diagnosis Patterns

When eval sub-scores reveal the problem:

| Pattern | Meaning | Strategic question |
|---------|---------|-------------------|
| d < c and d < v | Feature exists but delivers nothing | Should this feature exist at all? |
| c < d and c < v | Delivers value but fragile/ugly | Ticking time bomb or acceptable at this stage? |
| v < d and v < c | Works but users struggle | Does this block adoption? |
| d high, c high, v low | Well-built, well-designed, nobody cares | Kill it or reposition. |
| All sub-scores < 40 | Feature is half-baked | Focus here or kill it. Don't let it linger. |

## Honest Mode Toolkit

Questions that bypass the comfort zone:

- What are you avoiding? (Name it. The eval score that proves it exists.)
- What would a smart skeptic say after looking at the code for 10 minutes?
- If you had to bet $10,000 of your own money on this working, would you?
- What would you build if you started over today with what you know now?
- What's the most likely way this fails?
- What's the one thing you'd be embarrassed about if someone saw it?
- If this is the right product, why hasn't anyone else built it?
